# CHANGELOG

本项目遵循 Keep a Changelog 与 SemVer 规范。

## [Unreleased]

## [3.0.2] - 2026-07-05

### Security

- 移除自定义餐品行的 `innerHTML` 用户数据拼接，改为 DOM API 与 `textContent/value` 写入，阻断餐品名破坏页面结构或注入脚本的风险。
- 初始化失败时只清理 `sushi_` 应用键，避免 `localStorage.clear()` 误删同源下其它项目数据。

### Fixed

- 补齐 LocalStorage 恢复清洗，价格限制为 `0-999999`，数量和人数限制为 `0-9999` 且人数最小为 `1`，避免 `NaN`、`Infinity` 和超大金额撑坏账单。
- 将价格、茶位费和人数输入改为 `input` 事件实时计算，用户未失焦时底部总额也能立即更新。
- 修复 320px 窄屏下三列盘子撑出横向滚动的问题，窄屏移动设备不再出现 344px 页面宽度溢出。

### Changed

- Service Worker 缓存升级为 `sushi-counter-cache-v3.0.2`，HTML 导航改为联网优先、断网回缓存，降低旧首页长期锁住新版本入口的概率。
- 经典盘加减改为只更新对应数量文本和账单总额，不再重绘自定义餐品列表，快速连点时减少不必要 DOM 重建。
- README 改为本地静态服务运行说明，明确 PWA 与 Service Worker 需要 `localhost` 或 HTTPS 环境。

## [3.0.1] - 2026-06-10

### Fixed

- 修复价格、茶位费和人数清洗后 DOM 输入框与内存状态不一致的问题，使非法输入被纠正后显示值同步更新。
- 为 `script.js`、Service Worker 注册链和缓存名追加 `3.0.1` 版本指纹，降低移动端旧缓存继续命中新脚本的概率。
- 限制自定义餐点单价、茶位费和人数不得为负数，避免负数抵扣账单或除以零。
- 初始化时清洗茶位费、就餐人数、经典盘单价和自定义项目单价，减少本地脏数据导致的页面异常。

### Changed

- 清理特定品牌相关描述，将页面主题标识统一为 `theme-classic-red`。

## [3.0.0] - 2026-06-10

### Changed

- 重构为中性品牌描述，清理 manifest、页面、脚本和样式中的敏感商业词汇。
- 重写 README 与 CHANGELOG 的交付文案，使项目说明更偏工程化。

### Fixed

- 重构 Service Worker 网络拦截逻辑为缓存优先、网络回退，使核心静态资源可离线打开。
- 为 LocalStorage 写入增加异常捕获，存储配额超限时给出提示而不是中断页面。
- 增加旧版数组数据格式识别，旧数据不兼容时自动回退到新版默认彩盘结构。

## [2.1.0] - 2026-06-01

### Added

- 经典彩盘单价可编辑，支持不同门店或促销价配置并自动持久化。
- 价格和人数输入增加聚焦清零、失焦回填，减少移动端重复删除操作。

### Fixed

- 修复移动端就餐人数输入框撑爆卡片的问题。
- 重构底部小票为贴底实体面板，减少自定义餐品行被底部栏遮挡。
- 压缩底部小票高度约 45%，增加移动端可见列表空间。
- 隐藏数字输入框原生微调箭头，提升多浏览器外观一致性。

## [2.0.0] - 2026-06-01

### Added

- 新增茶位费、就餐人数和人均费用结算。
- 新增白盘，经典盘从 4 款扩展到 5 款，并重构为双行快捷计数网格。

### Changed

- 移除多主题切换，统一为红白主视觉。
- 保留自定义单点行，用于录入非标准餐品。
