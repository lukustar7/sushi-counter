# 寿司计数器 (Sushi Counter) v3.0.0

## 🎯 专业的日式回转寿司收银与人均结算计数面板

本计数器专为日式回转寿司用餐场景设计，提供图形化盘子计数、就餐参数（茶位费、人数）配置以及实时人均费用结算功能，旨在为用户提供清晰、规范、高效的就餐账单管理体验。

---

## ✨ 核心功能与技术设计

1. **🎨 统一的经典日式红白视觉设计**
   - 界面整体采用经典的日式红白主题，融入旋转传送带动态装饰线条。
   - 布局聚焦于核心计数业务，删除了冗余的视觉切换，确保了用户界面的直观与统一。

2. **🍽️ 图形化“双行五色圆形寿司盘”快捷计数网格**
   - 预设 5 款经典彩盘以配合不同定价等级的餐点：
     - **第一行**：白盘 (8元)、红盘 (10元)
     - **第二行**：银盘 (15元)、金盘 (20元)、黑盘 (28元)
   - **拟真圆形盘子绘制**：通过纯 CSS 渲染出带有拟物立体感与渐变盘缘的圆形寿司盘图形。
   - **快捷交互**：点击圆形盘身任意区域即可快速累加计数，盘子下方配有大触控的独立 `+` 与 `-` 按钮，便于精准调整。
   - **内置价格可直接编辑**：支持点击彩盘下方单价直接修改内置单价（如应对促销、调价或不同门店配置），修改后的单价自动持久化保存，账单随之实时更新。

3. **🍵 就餐配置面板 (茶位费与人数控制)**
   - **茶位费管理**：支持设置茶位费单价（默认 5 元/人），茶位费总额（`单价 × 就餐人数`）自动计入总额。
   - **就餐人数控制**：默认 1 人，支持通过加减按钮或数字键盘直接修改。

4. **💵 实时账单与“人均费用”结算**
   - 底部面板特化为小票样式，实时展现“总计”与“就餐人均”金额。
   - 采用严密的边界防呆计算，若就餐人数减少至 0 时，人均金额自动等同于总金额，成功避免了除零错误（如 `NaN` 或 `Infinity`），保障了数值显示的稳定性。

5. **🍜 横行自定义单点餐品区**
   - 针对非标准盘子的其他点单（如拉面、甜品、饮料等），支持点击底部 `+` 按钮动态添加横行，用户可自由输入名称、单价，并进行加减计数。

6. **🔋 物理按压反馈与数据持久化**
   - 所有交互按钮均配置了按压物理缩放弹性反馈，提升了操作流畅度与触屏反馈。
   - 所有的就餐配置、经典盘盘数、单价、自定义行数据均通过 `LocalStorage` 自动保存在浏览器本地，防止因页面刷新或意外退出导致的数据丢失。并且对数据保存逻辑进行了 `try-catch` 捕获异常处理，防止存储配额超限导致进程卡死崩溃。

7. **📲 PWA (Progressive Web App) 桌面安装与离线使用**
   - **添加到主屏幕**：配置了标准的 Web 描述清单 `manifest.json` 与矢量桌面图标 `icon.svg`，支持将本计数器作为独立应用安装到手机桌面，运行展现全屏无浏览器地址栏的原生化体验。
   - **Service Worker 离线运行**：利用 `sw.js` 实现核心静态资源（HTML、CSS、JS）的拦截与缓存管理，采用缓存优先（Cache First）的网络加载策略。在完全没有移动网络信号的环境下（如部分地下室门店）依然可以实现 0.1 秒内快速启动并正常计数结算。

## 🚀 部署与运行

本项目采用纯粹的 **Vanilla JS + Vanilla CSS** 原生无框架设计，极其轻巧，无任何第三方依赖。

直接使用任意浏览器打开项目根目录下的 **[index.html](file:///Users/luku/sushi-counter/index.html)** 即可运行。

### 📲 如何添加至手机主屏幕运行？
- **iOS 设备 (Safari 浏览器)**：
  1. 使用 Safari 浏览器打开计数器网址。
  2. 点击底部“分享”按钮（带向上箭头的方框）。
  3. 往下拉，选择 **“添加到主屏幕 (Add to Home Screen)”**。
  4. 点击“添加”即可。
- **Android 设备 (Chrome 浏览器)**：
  1. 使用 Chrome 打开计数器网址。
  2. 点击右上角“三点”菜单栏。
  3. 选择 **“安装应用 (Install App)”** 或 **“添加到主屏幕”** 即可。
